# test_cases.md — 정상 / 실패 / 경계 〔완료정의〕

> STEP 10에서 실제 결과를 채운다. v1 테스트 기준은 `엑셀 총판 업로드 → 대리점별 JPG 생성 → 다운로드 상태 표시`다.

| 유형 | 입력 | 기대 | 결과 |
|---|---|---|---|
| 정상 | 필수 컬럼이 모두 있는 월별 엑셀 총판 업로드 | 업로드 성공, 대리점별 데이터 DB화, 대시보드에 건수 표시 |  |
| 정상 | `더금융서비스` 확정 데이터 | 최종 포스터형 미리보기와 JPG 다운로드 버튼 표시 |  |
| 정상 | `A+에셋` 확정 데이터 | 시상값·상품별 지급률이 템플릿 지정 위치에 반영되고 JPG 생성 완료 표시 |  |
| 경계 | `인카금융서비스` 협의중 데이터 | 숫자는 비공개, JPG 다운로드는 제한, 상태만 표시 |  |
| 실패 | 필수 컬럼이 누락된 엑셀 업로드 | 업로드 실패 사유와 누락 컬럼 표시, 기존 확정본 훼손 없음 |  |
| 실패 | 숫자 칸에 문자 또는 빈 값 입력 | 해당 행 오류 표시, 승인·JPG 생성 제한 |  |
| 정상 | 카톡 공유용 홈페이지 URL 접속 | 월별 최신 프로모션 목록, 대리점 검색, 상태 필터, JPG 다운로드 영역 표시 |  |
| 정상 | 공유 홈페이지에서 `더금융서비스` 조회 | 해당 대리점 확정 포스터형 화면과 JPG 다운로드 버튼 표시 |  |
| 보안 | 승인 전 다운로드 URL 직접 접근 | 다운로드 차단, 승인 전 데이터 안내 표시 |  |
| 보안 | 승인 전 또는 협의중 프로모션 조회 | 목록에는 상태만 표시되고 숫자/JPG 다운로드는 제한 |  |
| 완료 | 업로드·검증·승인·JPG 다운로드 10회 반복 | 10회 모두 오류 없이 완료되면 PoC 완료 |  |
