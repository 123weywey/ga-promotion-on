# Agent 기획서 (PRD) — GA 프로모션ON

> 월별 3개 운영 파일(프로모션 총괄판, PPT 프로모션 장표 템플릿, 점포현황)을 업로드하면 하나의 홈페이지형 공유 URL에서 모든 사용자가 해당 월/대리점 프로모션을 확인하며 JPG를 다운로드하기 위한 PoC 문서. 실제 업무 효율, 일반 사용자 접근성, 간단한 운영 가능성을 우선 기준으로 둔다.

| 칸 | 내용 |
|---|---|
| **문제 정의** | 프로모션의 GA별 조건 차이와 반복적인 자료 재가공 때문에 현장 배포가 지연된다. 현재처럼 카톡에 파일을 반복 첨부하면 최신본·대상자·다운로드 여부가 섞여 확정 여부 문의와 지점장 추가업무가 반복된다. |
| **대상 사용자 / JTBD** | RM·영업추진P·지점장. "엑셀 총판에 확정 숫자만 입력하면 하나의 홈페이지에 전체 프로모션이 정리되고, 모든 사용자가 같은 URL에서 필요한 대리점 프로모션과 JPG를 바로 확인하게 하고 싶다." |
| **핵심 기능 1** | 월별 3개 파일 업로드(프로모션 총괄판·PPT 프로모션 장표 템플릿·점포현황) → 필수 컬럼/파일 형식 검증 → 월별 저장소 DB화 (입력 3개 파일 / 처리 스키마 검증·정규화 / 출력 업로드 결과·오류 목록) |
| **핵심 기능 2** | 공유 홈페이지 접속 → 월/대리점/상태별 목록 조회 → 협의중/확정 구분 → 확정 데이터 숫자와 미리보기 확인 |
| **핵심 기능 3** | 프로모션 총괄판의 확정 수치 + 점포현황의 지점/지사 매칭 + PPT 템플릿 배경 → 대리점별 최종 시책안내장 JPG 생성·다운로드 |
| **핵심 기능 4** | 공유 URL 고정 → 카톡에는 홈페이지 링크 1개만 배포 → 모든 사용자가 같은 페이지에서 필요한 프로모션 조회·다운로드 |
| **안 만들 것(보류·금지)** | 협상 판단 자동화, 확정 전 수치 공개, 외부 사용자 무제한 조회, 지점별 취향 커스터마이징, 상품전략까지 반영한 복잡한 안내장 자동화 |
| **입력 데이터** | 월별 프로모션 총괄판, 월별 PPT 프로모션 장표 템플릿, 월별 점포현황. v1 운영 입력은 이 3개 파일로 제한한다. |
| **출력 형식** | 단일 홈페이지형 공유 URL + 프로모션 목록 대시보드 + 대리점별 확정 포스터형 화면 + 최종 JPG 파일 다운로드 |
| **사람 승인 지점(HITL)** | 메인 유저가 업로드 결과와 협의 완료 수치를 검토·승인하고, 최종 배포 전 홈페이지 노출 및 안내장 JPG를 승인한다. 일반 사용자는 승인권한 없이 같은 URL에서 확정본을 확인한다. |
| **완료 정의** | 더미데이터로 대리점별 시책안내장 JPG가 생성되고, 다운로드 여부와 대시보드 상태가 표시되며, 총 10회 업로드·생성·다운로드 테스트가 오류 없이 수행되면 완료로 본다. |
| **D5 인계 메모** | 입력=월별 총괄판·PPT 템플릿·점포현황 3개 파일, 처리=검증→Supabase 저장→지점/대리점 매칭→홈페이지 대시보드 반영→안내장 렌더링, 출력=공유 URL에서 확인 가능한 프로모션 목록·대리점별 JPG 시책안내장·다운로드 상태 |
| **보류 이유(한 줄)** | 초기 버전은 정확한 월별 프로모션 안내장 JPG 구현이 목표이며, 지점장별 기호·상품전략 반영은 v1 안정화 후 검토한다. |

## 후보 검증 질문 5개
1. 월별 3개 파일만 올려도 현업이 반복 운영하기 쉬운 구조인가?
2. PPT 프로모션 장표 템플릿 위에 자동 반영해야 하는 숫자 위치와 항목은 고정 가능한가?
3. 협의중 상태에서 숫자 비공개와 확정 후 공개 규칙이 현장 문의를 줄이는가?
4. JPG 다운로드 파일명이 월·대리점·상태를 포함해 최신본 혼선을 줄이는가?
5. 하나의 공유 홈페이지 URL만으로 현장 사용자가 필요한 프로모션을 찾고 다운로드할 수 있는가?
