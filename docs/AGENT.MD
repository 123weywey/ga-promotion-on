# AGENT.MD — GA 프로모션ON 운영 기준

## 프로젝트 정의
`GA 프로모션ON`은 삼성생명 GA사업부의 월별 프로모션 협의 결과를 엑셀 총판으로 모으고, 하나의 홈페이지형 공유 URL에서 모든 사용자가 월별·대리점별 프로모션과 최종 시책안내장 JPG를 확인하도록 만드는 업무 효율화 프로젝트다.

## 주요 사용자
- Primary: RM, 영업추진P, GA사업부 프로모션 담당자
- Secondary: 지점장, GA영업관리자

## 핵심 흐름
1. RM 또는 영업추진P가 월별 운영 파일 3개(프로모션 총괄판, PPT 프로모션 장표 템플릿, 점포현황)를 준비한다.
2. 3개 파일을 업로드하면 총괄판 수치, PPT 템플릿, 점포현황 매칭 정보가 검증되고 월별 데이터로 저장된다.
3. 대시보드에서 협의중/확정 상태, 업로드 오류, JPG 생성 여부를 확인한다.
4. 메인 유저가 확정 수치와 최종 안내장을 검토·승인한다.
5. 카톡에는 파일 여러 개가 아니라 홈페이지 URL 하나만 전달한다.
6. 지점장 또는 GA영업관리자는 같은 URL에 접속해 월/대리점/상태별 프로모션을 확인하고 필요한 JPG를 다운로드한다.

## 입력 / 처리 / 출력
- 입력: 월별 프로모션 총괄판, 월별 PPT 프로모션 장표 템플릿, 월별 점포현황
- 처리: 3개 파일 검증 → Supabase 월별 저장 → 점포현황 기준 지점/대리점 매칭 → 총괄판 수치 상태 판정 → 홈페이지 대시보드 반영 → 템플릿 렌더링 → 승인 → JPG 생성·다운로드 기록
- 출력: 단일 홈페이지형 공유 URL + 프로모션 목록 대시보드 + 대리점별 최종 포스터형 화면 + JPG 파일 다운로드

## 운영 원칙
- 한 화면은 `그 달의 최종 장표 1장` 기준으로 구성한다.
- 월별 이력 비교보다 `이번 달 확정값`과 `다운로드 가능 여부`를 우선 보여준다.
- 카톡 공유는 파일 첨부 반복이 아니라 `홈페이지 URL 1개 + 필요 시 안내 문구`로 단순화한다.
- 일반 직원도 월별 3개 파일 업로드만으로 운영할 수 있게 구조를 단순하게 유지한다.
- 실제 업무 시간이 줄어드는지, 반복 문의와 수작업 장표 제작이 줄어드는지를 우선 판단한다.

## 상태 원칙
- `협의중`: 상태만 표시하고 숫자와 JPG 다운로드는 비공개 또는 제한
- `확정`: 승인된 시상값과 상품별 지급률을 포스터형 화면과 JPG에 동일하게 반영
- `오류`: 필수 컬럼 누락, 숫자 형식 오류, 중복 대리점 등 원인을 대시보드에 표시

## 보안 / 검수 원칙
- 확정 전 수치는 외부 공개하지 않는다.
- 실제 대외 공유, 확정 전환, 권한 부여는 사람이 최종 확인한다.
- 일반 사용자는 승인권한을 갖지 않으며, 홈페이지에서는 승인된 확정본만 확인·다운로드하게 한다.
- 최신본 혼선을 줄이기 위해 JPG 파일명은 기준월·대리점명·생성일을 포함한다.

## 완료 기준
- 월별 3개 파일 업로드가 정상 검증된다.
- 각 대리점별 시책안내장 JPG가 생성되고 다운로드 여부가 기록된다.
- 대시보드에서 협의중/확정/오류/승인 상태가 구분된다.
- 총 10회 업로드·생성·다운로드 테스트가 오류 없이 수행된다.
