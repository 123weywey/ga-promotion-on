# 프로모션 ON

`프로모션 ON`은 삼성생명 GA사업부의 월별 프로모션 협의 결과를 하나의 공유 홈페이지로 정리하고, 대리점별 최종 시책안내장 미리보기와 JPG/PPT 다운로드까지 연결하기 위한 운영형 대시보드입니다.  
기존처럼 카카오톡이나 메일로 파일을 반복 첨부하는 방식 대신, 모든 사용자가 같은 URL에서 같은 기준월의 최신 확정본을 확인하도록 만드는 것이 이 프로젝트의 핵심 목적입니다.

## 배포 URL

- 운영 URL: https://ga-promotion-on.vercel.app/docs/final-monthly-promotion-dashboard

## 제품 개요

GA 프로모션 운영에서는 대리점별 조건 차이, 확정 여부 확인, 장표 재가공, 최신본 혼선 때문에 배포 지연과 반복 문의가 자주 발생합니다.  
`프로모션 ON`은 이 문제를 줄이기 위해 월별 운영 파일 3개만 업로드하면, 지점별 프로모션 현황과 대리점별 최종 안내장 화면을 자동으로 연결하는 구조로 설계되었습니다.

이 프로젝트는 단순한 파일 보관함이 아니라 다음 업무를 하나의 흐름으로 묶는 운영 화면입니다.

- 월별 프로모션 총괄판의 확정 수치 관리
- 점포현황 기준 지역단/지점/대리점 매칭
- PPT 템플릿 기반 최종 프로모션 장표 렌더링
- 지점별 협의중/확정 현황 조회
- 최종 JPG/PPT 다운로드 및 공유

## 해결하려는 문제

- GA별 조건 차이와 반복적인 장표 재가공 때문에 현장 배포가 늦어짐
- 카톡/메일로 파일을 여러 번 전달하면서 최신본과 확정본 구분이 어려워짐
- 지점장, RM, 영업추진P 사이에서 확정 여부와 대상 대리점 문의가 반복됨
- 확정된 숫자를 다시 장표에 옮기고 JPG로 만드는 수작업이 큼
- 같은 월의 프로모션이라도 지점별 담당 대리점 범위가 달라 화면에서 즉시 구분이 필요함

## 주요 사용자

- Primary: RM, 영업추진P, GA사업부 프로모션 담당자
- Secondary: 지점장, GA영업관리자
- 운영 검토자: 승인 전 노출 여부, 기준 데이터 정합성, 다운로드 가능 상태를 확인하는 내부 운영 담당자

각 사용자가 기대하는 결과는 다르지만 공통 목표는 같습니다.  
`엑셀 총괄판에 확정 수치만 반영하면, 하나의 홈페이지에서 모든 사용자가 최신 상태와 최종 장표를 같은 기준으로 본다`는 점입니다.

## 핵심 운영 방식

`프로모션 ON`은 매월 아래 3개 파일만으로 작동하도록 설계되어 있습니다.

1. `프로모션 총괄판`
대리점별 코드, 담당 여부, 최종확정여부, 상품별 지급률/시상값 등 월별 프로모션 숫자 데이터의 기준 파일입니다.

2. `점포현황`
지역단, 지점, 대리점, 조직번호, 등록번호 등 조직 편제와 매칭 기준이 되는 파일입니다.  
지점을 선택했을 때 어떤 대리점이 기본 대상이어야 하는지는 이 파일을 우선 기준으로 삼습니다.

3. `PPT 프로모션 장표 템플릿`
최종 프로모션 미리보기와 다운로드 결과물의 배경/레이아웃/양식이 되는 템플릿 파일입니다.  
총괄판 숫자는 이 템플릿 위에 반영되어 대리점별 최종 프로모션 장표로 생성됩니다.

## 입력 → 처리 → 출력

### 입력

- 월별 프로모션 총괄판 업로드
- 월별 점포현황 업로드
- 월별 PPT 프로모션 장표 템플릿 업로드

### 처리

- 필수 컬럼과 파일 형식을 검증합니다.
- 점포현황 기준으로 지역단/지점/대리점 매칭 구조를 만듭니다.
- 총괄판 코드와 점포현황의 등록번호/조직번호를 연결해 대상 대리점을 판정합니다.
- 총괄판의 값이 있는 대리점은 확정 또는 승인 가능 대상으로 반영하고, 값이 없는 대상은 협의중으로 표시합니다.
- 템플릿 장표 위에 확정 숫자를 배치해 최종 프로모션 미리보기와 다운로드 결과물을 생성합니다.
- 월별 운영 데이터는 Supabase를 기준 저장소로 관리합니다.

### 출력

- 단일 공유 URL 기반 지점별 프로모션 현황 대시보드
- 지역단/지점/상태/대리점 검색이 가능한 조회 화면
- 대리점별 최종 프로모션 미리보기
- JPG 다운로드
- PPT 다운로드

## 화면에서 제공하는 핵심 기능

- 지점별 총 지사 / 확정 완료 / 협의중 현황 표시
- 지역단 → 지점 → 상태 → 대리점명 검색 흐름 제공
- 검색창 1글자 이상 입력 시 대리점 예시 추천 표시
- 확정된 대리점만 미리보기, JPG, PPT 다운로드 활성화
- 관리자 모드에서 월별 운영 파일 3종 업로드
- 같은 URL을 PC와 모바일에서 공통 사용

## 운영 원칙

- 한 달 기준으로 `그 달의 최종 장표 1장`을 빠르게 확인하는 것이 우선입니다.
- 월별 비교보다 `이번 달 최신 확정본`과 `즉시 다운로드 가능 여부`를 더 중요하게 봅니다.
- 일반 직원도 3개 파일 업로드만으로 운영할 수 있게 구조를 단순하게 유지합니다.
- 지점별 커스터마이징이나 복잡한 상품전략 자동화는 v1 범위에서 제외합니다.
- 카톡에는 파일 여러 개 대신 홈페이지 URL 하나를 전달하는 운영을 지향합니다.

## 상태 및 승인 기준

- `협의중`: 점포현황상 대상 대리점이지만 총괄판 수치가 아직 비어 있거나 확정되지 않은 상태
- `확정`: 총괄판 수치가 채워지고 최종 장표 반영이 가능한 상태
- `오류`: 필수 컬럼 누락, 중복, 숫자 형식 이상 등으로 정상 반영이 어려운 상태

최종 노출과 배포는 사람 승인 절차를 전제로 합니다.  
일반 사용자는 승인권한 없이 확정본만 조회·다운로드하고, 메인 운영자는 업로드 결과와 장표 반영값을 확인한 뒤 배포 여부를 판단합니다.

## 저장 및 배포 원칙

- 운영 데이터는 Supabase를 기준 저장소로 관리합니다.
- 월별 업로드 원본 파일(`.xlsx`, `.pptx`)과 업로드 기반 생성 데이터는 GitHub에 저장하지 않습니다.
- GitHub 저장소는 화면 소스와 운영 로직 관리용으로만 사용합니다.
- 공개 접속은 Vercel 배포 URL을 기준으로 제공합니다.

## 기대 효과

- 반복적인 장표 수정 시간을 줄입니다.
- 지점별 최신 확정본 혼선을 줄입니다.
- 확정 여부 문의와 대상 대리점 문의를 줄입니다.
- 하나의 URL로 공유 체계를 단순화합니다.
- 다운로드 결과물과 화면 미리보기가 같은 기준 숫자를 보도록 운영 일관성을 높입니다.
